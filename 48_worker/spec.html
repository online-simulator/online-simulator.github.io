共通化するjob定義 		mainとworkerそれぞれでimportする共通のスクリプトにjobを定義する
mainから受け取ったdataを処理した後、同様のdataを返却する
いずれかのエラー発生場所を特定するためにwindowオブジェクトの存在可否を判定する
workerからはmainのwindowやdocumentにアクセスできないことに留意する
そのため、ここで定義するjobは並列化向きのメディア処理や数値計算が一般的と考えられる 

My_entry.job_imported = function(_data){
  var out = null;
  var isWorker = (typeof window === "undefined");
  try{
    out = Math[_data.sw_job](_data.in);
  }
  catch(e){
    out = e.message;
    out += (isWorker)? "@worker": "@main";
    throw new Error(out);
  }
  _data.out = out;
  return _data;
};

handler_worker
実行処理 		上記のself.handler_worker.runの関数を以下に示す
WebWorkers非対応かロック状態の場合は即座にfalseを返却する
それ以外の場合はロック後、self.workerの中断terminateを判定をしながら
postMessageで外部スクリプトに繰り返しdataを送信する
なお、繰り返しnew Workerを宣言する必要はないことに留意する 

My_entry.handler_worker.prototype.run = function(arr_data){
  var self = this;
  if(!(self.hasWorker) || self.isLocked) return false;
  self.isLocked = true;
  arr_data.forEach(function(data){
    if(self.worker){
      self.worker.postMessage(data);
    }
  });
  return self;
};

中断・終了処理 		handler_workerの中断・終了処理の関数を以下に示す
self.worker.terminateを呼び出して全スレッドを強制終了する
また、メモリを解放するself.worker = null
さらに、ロック状態を解除する
なお、workerをteminateした時点で再使用できないため、

My_entry.handler_worker.prototype.terminate = function(){
  var self = this;
  if(self.worker){
    self.worker.terminate();
  }
  self.worker = null;
  self.isLocked = false;
  return self;
};
